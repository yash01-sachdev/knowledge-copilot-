# Deployment Notes

Knowledge Copilot is meant to deploy as a local-first AI app, not as an API-wrapper demo.

The recommended public deployment is a self-hosted stack:

- Next.js frontend
- FastAPI backend
- SQLite database on persistent disk
- ChromaDB vector index on persistent disk
- Ollama running beside the backend

## What `/data` Means

`/data` is only a folder inside the backend Docker container.

The app stores its SQLite database at:

```text
/data/knowledge_copilot.db
```

It also stores the Chroma vector index at:

```text
/data/chroma
```

Docker mounts a persistent volume there:

```yaml
volumes:
  - knowledge_copilot_data:/data
```

That means both the SQLite note store and the Chroma vector index survive container rebuilds and redeploys. It is not an external data platform, and it is not where you put private notes for the portfolio demo. For a public resume demo, load demo notes only.

## Recommended Resume Deployment

Use one VPS and run everything with Docker Compose.

Recommended providers:

```text
KNOWLEDGE_COPILOT_VECTOR_STORE=chroma
KNOWLEDGE_COPILOT_CHROMA_PATH=/data/chroma
KNOWLEDGE_COPILOT_EMBEDDING_PROVIDER=ollama
KNOWLEDGE_COPILOT_EMBEDDING_MODEL=nomic-embed-text
KNOWLEDGE_COPILOT_ANSWER_PROVIDER=local
KNOWLEDGE_COPILOT_ANSWER_MODEL=heuristic
KNOWLEDGE_COPILOT_RERANKER_PROVIDER=local
KNOWLEDGE_COPILOT_RERANKER_MODEL=heuristic
OLLAMA_BASE_URL=http://ollama:11434
```

This keeps the project local-first:

- no OpenAI key
- no hosted model API
- Chroma handles semantic vector search inside your own stack
- embeddings generated by your own Ollama service
- answers composed locally from retrieved evidence

## VPS Steps

1. Rent a VPS.

Use Ubuntu 22.04 or 24.04. A 4 GB RAM machine can run the fast embedding-only demo. An 8 GB RAM machine is more comfortable.

2. Install Docker and Docker Compose.

3. Clone the repo.

```bash
git clone https://github.com/YOUR_USERNAME/knowledge-copilot.git
cd knowledge-copilot
```

4. Create `.env` next to `docker-compose.yml`.

```text
NEXT_PUBLIC_API_BASE_URL=https://api.your-domain.com
KNOWLEDGE_COPILOT_CORS_ORIGINS=https://your-domain.com
KNOWLEDGE_COPILOT_DB_PATH=/data/knowledge_copilot.db
KNOWLEDGE_COPILOT_VECTOR_STORE=chroma
KNOWLEDGE_COPILOT_CHROMA_PATH=/data/chroma
KNOWLEDGE_COPILOT_EMBEDDING_PROVIDER=ollama
KNOWLEDGE_COPILOT_EMBEDDING_MODEL=nomic-embed-text
KNOWLEDGE_COPILOT_ANSWER_PROVIDER=local
KNOWLEDGE_COPILOT_ANSWER_MODEL=heuristic
KNOWLEDGE_COPILOT_RERANKER_PROVIDER=local
KNOWLEDGE_COPILOT_RERANKER_MODEL=heuristic
OLLAMA_BASE_URL=http://ollama:11434
```

5. Start the stack.

```bash
docker compose --env-file .env up --build -d
```

6. Pull the Ollama embedding model.

```bash
docker compose exec ollama ollama pull nomic-embed-text
```

7. Verify the backend.

```bash
curl http://localhost:8000/health
```

Expected response:

```json
{"status":"ok"}
```

8. Load demo notes.

```bash
curl -X POST http://localhost:8000/api/demo/load
```

9. Put HTTPS in front.

Use Caddy or Nginx:

- `https://your-domain.com` -> frontend container on port `3000`
- `https://api.your-domain.com` -> backend container on port `8000`

After HTTPS is live, rebuild the frontend with the final API URL:

```bash
docker compose --env-file .env up --build -d web
```

`NEXT_PUBLIC_API_BASE_URL` is a frontend build-time value, so the frontend must be built with the real backend URL.

## Simpler But Weaker Deployment

If you do not want to run Ollama on the VPS, use the fully local fallback:

```text
KNOWLEDGE_COPILOT_VECTOR_STORE=local
KNOWLEDGE_COPILOT_EMBEDDING_PROVIDER=local
KNOWLEDGE_COPILOT_ANSWER_PROVIDER=local
KNOWLEDGE_COPILOT_RERANKER_PROVIDER=local
```

That still uses no external AI APIs, but retrieval quality is weaker than Ollama embeddings.

## Optional Full Local LLM Answers

If the server has enough RAM/CPU or a GPU, you can also use Ollama for answer generation:

```text
KNOWLEDGE_COPILOT_ANSWER_PROVIDER=ollama
KNOWLEDGE_COPILOT_ANSWER_MODEL=qwen2.5:3b
```

Then pull the model:

```bash
docker compose exec ollama ollama pull qwen2.5:3b
```

This is more impressive, but slower on cheap CPU-only servers. For a resume demo, Ollama embeddings plus local grounded answers is the better default.
